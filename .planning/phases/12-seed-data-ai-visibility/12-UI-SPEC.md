## The Only UI Change: Default-Expanded Seed Row

**What changes:** `ListingsTable.tsx` — `expandedUrl` state initialization.

**Current behavior:** `useState<string | null>(null)` — no row is expanded on load. User must click a row to expand the AgentDecisionPanel.

**Required behavior (DEMO-01 success criterion 3):** The first seed listing row is expanded on mount — AgentDecisionPanel is visible without any click.

**Implementation contract:**

```typescript
// dashboard/src/components/ListingsTable.tsx

// Add this constant above the component — seed URLs are canonical and must match LISTINGS.md exactly
const SEED_URLS = [
  'https://ducket.seed/listing/scalping-001',
  'https://ducket.seed/listing/scam-001',
  'https://ducket.seed/listing/counterfeit-001',
  'https://ducket.seed/listing/legitimate-001',
];

// Change this line only:
// BEFORE: const [expandedUrl, setExpandedUrl] = useState<string | null>(null);
// AFTER:
const [expandedUrl, setExpandedUrl] = useState<string | null>(SEED_URLS[0]);
```

**Why only the first seed row:** Opening all 4 simultaneously would stack 4 AgentDecisionPanels on first load — overwhelming for a demo. One expanded row demonstrates the capability; judges click to see others. Source: 12-RESEARCH.md Open Question 1.

**Visual result of this change:**
- On first load, the SCALPING_VIOLATION seed row is expanded
- AgentDecisionPanel renders with `bg-bg-card/50 border border-brand-primary/40 rounded-lg p-4`
- 91% confidence bar fills most of the ConfidenceBar component
- Reasoning text (65–90 words) is fully visible at `text-sm` in `text-foreground`
- Etherscan link renders in `text-brand-accent underline font-mono text-xs`
- Action Taken "slash" renders in `text-warn-red font-semibold`

**No other changes to ListingsTable.tsx are required.**

## AgentDecisionPanel Content Contract

The AgentDecisionPanel is already implemented. It reads `Classification` fields and renders them. The UI contract for Phase 12 is that the data in each seed case file produces readable, informative output in the existing component.

### Minimum content requirements per seed listing

| Field | Requirement | Source |
|-------|------------|--------|
| `category` | Exact uppercase enum string: `SCALPING_VIOLATION`, `LIKELY_SCAM`, `COUNTERFEIT_RISK`, or `LEGITIMATE` | 12-RESEARCH.md Pitfall 1 |
| `confidence` | Integer 0–100, rendered as `{value}%` by ConfidenceBar | 12-RESEARCH.md seed plan |
| `reasoning` | 50+ words; references at least 2 of: priceDeltaPct, faceValue, account age, eventName, section name | REQUIREMENTS.md DEMO-01 + 12-RESEARCH.md Pitfall 2 |
| `classificationSource` | `"demo-seed"` — rendered as metadata label in `text-muted-foreground text-sm` | 12-RESEARCH.md case file format |
| `actionTaken` | One of: `"release"`, `"refund"`, `"slash"` — must use `string.includes()` matching values | 12-RESEARCH.md Pitfall 4 |
| `etherscanLink` | Valid Sepolia Etherscan URL — rendered as `text-brand-accent underline font-mono text-xs break-all` | AgentDecisionPanel component |

### AgentDecisionPanel layout (unchanged — existing component)

```
[Category label (uppercase tracking-wide, muted)] [Confidence label]
[Badge component]                                  [ConfidenceBar component]

[Reasoning label (uppercase tracking-wide, muted)]
[reasoning text — text-sm text-foreground, 50+ words]

[Classification Source label]   [Action Taken label]
[source value — muted-fg sm]    [action value — colored sm font-semibold]

[divider — border-brand-primary/20]
[On-Chain Evidence label]
[etherscan link — brand-accent underline mono xs break-all]
```

**Action Taken color mapping** (existing `actionColor` logic in AgentDecisionPanel):

| actionTaken value | Color class |
|------------------|-------------|
| `"release"` | `text-success` |
| `"escrow_deposit"` (old) | `text-warn-red` — do NOT use this in seed data |
| anything else | `text-foreground` |

**Note:** The current `actionColor` logic in AgentDecisionPanel.tsx only handles `"release"` and `"escrow_deposit"` explicitly. For Phase 12 seed data using `"slash"` and `"refund"`, those values fall through to `text-foreground` (white). This is acceptable for the demo — the action text is still readable. If `"slash"` should render red, the executor may update `actionColor` to include it: `actionTaken.includes('slash') ? 'text-warn-red' : actionTaken.includes('refund') ? 'text-warn-yellow' : actionTaken === 'release' ? 'text-success' : 'text-foreground'`.
