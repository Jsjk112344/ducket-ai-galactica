## Spacing Scale

Standard 8-point scale. Source: default (no override specified in upstream artifacts).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, trust badge inline padding (`gap-1.5`, `px-1`) |
| sm | 8px | Compact element spacing, badge horizontal pad (`px-2 py-1`) |
| md | 16px | Default element spacing, card internal padding (`p-4`) |
| lg | 24px | Section padding, card large padding (`p-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Trust badge pill: `px-3 py-1` (12px / 4px) — intentionally compact to fit 3 badges in one row without scrolling
- Tab bar buttons: `px-6 py-2` (24px / 8px) — wider tab targets match existing App.tsx pattern
- Touch targets: not applicable (web-first, desktop viewport assumed per REQUIREMENTS.md out-of-scope: mobile responsive)

## Typography

Source: BRAND-02 (REQUIREMENTS.md) + reference_ducket_brand.md + RESEARCH.md Standard Stack.

| Role | Font | Size | Weight | Line Height | Applied To |
|------|------|------|--------|-------------|-----------|
| Display | Outfit Variable | 28px | 700 | 1.2 | Page title `<h1>` — "Ducket" brand header |
| Heading | Outfit Variable | 20px | 700 | 1.2 | Section headings `<h2>`, tab section titles |
| Body | Inter Variable | 16px | 400 | 1.5 | Default text, table cell content, descriptions |
| Caption | Inter Variable | 12px | 400 | 1.4 | Metadata labels, timestamps, stat card labels, uppercase tracking labels (`text-xs uppercase tracking-wide`), trust badge text, wallet addresses, tx hashes, footer |

**Weights in use: exactly 2** — 400 (regular) and 700 (bold). No other weights.

**Note on Label and Mono roles:** The previously separate 14px Label and 12px Mono roles are consolidated into the 12px Caption row. Metadata labels and timestamps use `text-xs` (12px, weight 400). Wallet addresses and tx hashes use `font-mono` at the same 12px size — this is a font-family variant, not a separate size or weight.

**Heading application:** Add `@layer base { h1, h2, h3, h4, h5, h6 { font-family: var(--font-family-heading); } }` to `index.css`. This applies Outfit Variable to all heading elements without requiring a class on each.

**Font family values:**
- `--font-family-heading: 'Outfit Variable', ui-sans-serif, system-ui, sans-serif;`
- `--font-family-sans: 'Inter Variable', ui-sans-serif, system-ui, sans-serif;`

Note: font family must be `'Outfit Variable'` (with "Variable" suffix), not `'Outfit'`. The @fontsource-variable package registers the font under this name.

## Color

Source: BRAND-01 (REQUIREMENTS.md) + reference_ducket_brand.md + RESEARCH.md Pattern 1.

### 60/30/10 Split

| Role | Hex | HSL | Usage |
|------|-----|-----|-------|
| Dominant (60%) | `#0a0714` | `263 50% 8%` | Page background (`bg-bg-primary`), `--background` |
| Secondary (30%) | `#130f1f` | `263 50% 10%` | Cards, tab content area, panels (`bg-bg-card`), `--card` |
| Accent (10%) | `#F5C842` | `45 90% 61%` | See reserved-for list below |
| Destructive | `#EF4444` | `0 75% 55%` | SCALPING_VIOLATION badge only; no destructive user actions exist in this phase |

**Accent `#F5C842` reserved for:**
1. Trust badge text color and icon tint
2. On-chain evidence links in AgentDecisionPanel (replaces current `text-accent` on `<a>`)
3. Stat card metric values (replaces current `text-accent` on stat numbers in StatCard)
4. Active tab bar indicator text (text only, not background — use `bg-brand-primary` for active tab bg)
5. "Sepolia Testnet" pill text in EscrowStatus

**Accent is NOT used for:** tab bar backgrounds, card backgrounds, borders, body text, labels, or any element outside the 5 items listed above.

**Primary brand purple `#3D2870` used for:**
- Active tab bar background (`bg-brand-primary`)
- Trust badge background tint (`bg-brand-primary/20`) and border (`border-brand-primary/40`)
- shadcn `--primary` variable (Button default variant background)

**Supporting palette (semantic — not brand accent):**

| Token | Hex | Usage |
|-------|-----|-------|
| `--color-success` | `#10B981` | LEGITIMATE badge, positive state indicators |
| `--color-warn-red` | `#EF4444` | SCALPING_VIOLATION badge |
| `--color-warn-orange` | `#F97316` | LIKELY_SCAM badge |
| `--color-warn-yellow` | `#EAB308` | COUNTERFEIT_RISK badge (existing — do not replace with #F5C842) |
| `--color-muted-fg` | `hsl(263 20% 60%)` | Metadata labels, timestamps, secondary text (replaces all `text-gray-400`, `text-gray-500`) |

**Accent collision fix — required actions:**

The existing codebase uses `text-accent`, `bg-accent`, `border-accent` for the old indigo `#6366F1`. After rebrand, `--color-accent` maps to yellow `#F5C842`. All existing usages must be audited and remapped:

| File | Line | Current class | New class |
|------|------|---------------|-----------|
| `App.tsx` | ~42 | `bg-accent text-white` (active tab) | `bg-brand-primary text-white` |
| `EscrowStatus.tsx` | ~21 | `text-accent` (stat value) | `text-brand-accent` |
| `EscrowStatus.tsx` | ~50 | `bg-accent/20 text-accent` (Sepolia pill) | `bg-brand-primary/20 text-brand-accent` |
| `AgentDecisionPanel.tsx` | ~33 | `border-accent/20` | `border-brand-primary/40` |
| `AgentDecisionPanel.tsx` | ~76 | `border-accent/10` | `border-brand-primary/20` |
| `AgentDecisionPanel.tsx` | ~83 | `text-accent` (tx link) | `text-brand-accent` |
| `WalletInspector.tsx` | ~36 | `bg-accent text-white` (WDK badge) | `bg-brand-primary text-white` |
| `ListingsTable.tsx` | ~112 | `border-accent/20` | `border-brand-primary/20` |

Also replace all `text-gray-*` and `bg-gray-*` with token equivalents — no default Tailwind gray visible in primary UI areas (BRAND-01 success criterion).
