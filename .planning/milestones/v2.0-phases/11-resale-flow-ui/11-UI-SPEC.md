Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label/value gaps |
| sm | 8px | Form field internal padding, gap between label and input |
| md | 16px | Card content padding (`p-4`), form grid gap, default element spacing |
| lg | 24px | Card content padding on settlement step (`p-6`), section spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch target minimum 44px height on all interactive buttons (enforced by shadcn Button `h-9` default + text padding — sufficient for demo web context)
- Step indicator strip uses `py-2` (8px vertical) with `flex-1` horizontal expansion

Source: Tailwind v4 default spacing + existing component patterns from `EscrowStatus.tsx`, `AgentDecisionPanel.tsx`.

## Typography

| Role | Font | Size | Weight | Line Height |
|------|------|------|--------|-------------|
| Body | Inter Variable | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | Inter Variable | 12px (`text-xs`) | 400 (regular) | 1.4 — uppercase tracking-wide for field labels |
| Heading | Outfit Variable | 18px (`text-lg`) | 600 (semibold) | 1.2 |
| Display | Outfit Variable | 20px (`text-xl`) | 600 (semibold) | 1.2 — settlement outcome label only |

Typography rules for this phase:
- Step content headings: `text-lg font-heading font-semibold text-white` — matches existing `AgentDecisionPanel` heading pattern
- Settlement outcome label: `text-xl font-semibold` in outcome color (green/yellow/red)
- Field labels: `text-xs text-muted-foreground uppercase tracking-wide` — matches existing component patterns
- Wallet address / Etherscan links: `font-mono text-xs break-all` — ensures long hex strings wrap cleanly
- Body copy / reasoning text: `text-sm text-foreground` — minimum 50 words for RESALE-03 reasoning field

Source: `AgentDecisionPanel.tsx` patterns + `index.css` font declarations confirmed live.

## Interaction States

### Step Indicator Strip
- 4 steps: List (1), Lock (2), Verify (3), Settle (4)
- Horizontal flex strip, `flex gap-2`, each step `flex-1`
- Navigation is linear — user cannot skip forward; can only advance via explicit button action
- No back-navigation in this phase (demo is forward-only)

### ListingForm (Step 1)
| State | Visual |
|-------|--------|
| Default | Pre-filled with FIFA World Cup 2026 seed values |
| Submitting | Button shows "Submitting..." + `disabled` attribute |
| Error | Not applicable — form validates required fields via HTML5 `required` attribute only |
| Success | `onSubmit` resolves → hook sets step to 2 |

Pre-filled default values (seed for demo):
- eventName: `FIFA World Cup 2026 — USA vs England`
- section: `Category 2`
- quantity: `2`
- price: `350`
- faceValue: `120`

### BuyerLockStep (Step 2)
| State | Visual |
|-------|--------|
| Before lock | "Lock 10 USDT in Escrow" button — full width, primary style |
| Locking | "Locking 10 USDT..." + `disabled` attribute — spinner not required (text change sufficient) |
| Lock complete | Button hidden; success text `text-success`; Etherscan link appears; "Proceed to AI Verification" button |
| Lock failed | Not explicitly handled — mock fallback in API ensures lock always succeeds |

**Critical:** Do not auto-advance after lock — user must see the Etherscan link and click "Proceed to AI Verification" manually.

### VerifyStep (Step 3)
| State | Visual |
|-------|--------|
| Default | AI avatar badge + heading + full AgentDecisionPanel + "View Settlement Outcome" button |
| Always populated | Classification injected server-side at listing creation — never null at step 3 |
| Reasoning length | Minimum 50 words, references `eventName`, `section`, `price`, `faceValue` |

### SettleStep (Step 4)
| State | Visual |
|-------|--------|
| Release | Green outcome card (`bg-success/10 border-success/30`), label "RELEASED to seller" |
| Refund | Yellow outcome card (`bg-warn-yellow/10 border-warn-yellow/30`), label "REFUNDED to buyer" |
| Slash | Red outcome card (`bg-warn-red/10 border-warn-red/30`), label "SLASHED to bounty pool" |
| Pending | Muted card (`bg-bg-card border-border`), label "PENDING settlement" |
| Etherscan link | Always shown below outcome card — uses `EtherscanLink` component |
| No reset button | Out of scope — demo is a single-pass forward flow |

## Page Layout

### Tab Placement
- Add "Resale Flow" as the **first tab** in `App.tsx` TABS array
- Tab id: `'resale'`, label: `'Resale Flow'`
- Existing tabs: Listings, Escrow, Wallet — shifted right
- `ResaleFlowPanel` renders as full tab content under existing layout shell

### ResaleFlowPanel Layout
```
[Step indicator: 4 columns, flex, full width]
[Step content area: single Card, full width]
  Step 1: ListingForm — 2-column grid for fields, full-width submit button
  Step 2: BuyerLockStep — single column, wallet address mono text, lock button, Etherscan link
  Step 3: VerifyStep — AI badge header + full-width AgentDecisionPanel + advance button
  Step 4: SettleStep — outcome card (color-coded) + Etherscan link
```

### Grid used in ListingForm
- eventName: `col-span-2` (full row)
- section + quantity: `col-span-1` each (2-column row)
- price + faceValue: `col-span-1` each (2-column row)
- submit button: `col-span-2` (full row)
- Grid: `grid grid-cols-2 gap-4`

## Accessibility

- All form inputs have associated `<Label>` with `htmlFor` matching `id` — enforced by shadcn Label + Input pattern
- All external links use `rel="noopener noreferrer"` — enforced by `EtherscanLink` component
- Buttons use `disabled` attribute during loading states — handled by shadcn Button
- Color is not the only indicator — outcome labels include text ("RELEASED", "REFUNDED", "SLASHED") in addition to card background color
- Step indicator uses text numbers + labels — not icon-only
